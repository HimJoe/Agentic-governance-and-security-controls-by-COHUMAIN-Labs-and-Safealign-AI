The thesis

For an autonomous agent, governance and security are the same question

Traditional assurance keeps them apart — different teams, different tools. Agentic AI collapses them: the autonomy that creates the governance question (what may the agent decide and do?) is the same autonomy that creates the security question (what can a compromised agent be made to do?).

One standard has to answer both — or it answers neither.

This standard is the catalog of what to control and why, unifying the governance spine (accountability, oversight, lifecycle) with the security spine (the adversarial and runtime defence of autonomous systems) so a single, defensible, regulator-ready control set can sit on any agent platform.

01

The failure mode inverts

Traditional IT fails by stopping. An agent fails by continuing — fully operational, acting wrongly, at machine speed, across systems before a human reacts. The safe state is "stopped," not "restored."

02

The model is the attack surface

Prompt injection is an attack written in plain language inside data — invisible to code scanning. The agent's own reasoning becomes the exploit path.

03

Orchestration is the risk, not skill

Across 832 real-world malicious actors, the highest-risk operations were set apart not by skill but by AI agents autonomously chaining attack stages — a behaviour today's threat frameworks can't yet name.

Grounded in real-world threat data

Defending against the weaponised agent

Each security control is mapped to the MITRE ATT&CK techniques a hijacked agent would actually execute — drawn from Anthropic's analysis of real AI-enabled cyber operations.

The differentiator is orchestration, not sophistication

Anthropic mapped one year of AI-enabled attacks — 832 actors, 13,873 observations — onto MITRE ATT&CK. The highest-risk operation used a technique count comparable to medium-risk actors, yet scored maximum risk. What set it apart: an AI agent autonomously orchestrating the killchain via tool-connected scaffolding.

That behaviour — autonomous, multi-step, AI-directed execution — has no ATT&CK ID yet. That gap is the reason the Safety and Security pillars carry dedicated autonomy, multi-agent, and threat-modeling controls.

Adoption

Three conformance tiers

A graduated path — adopt the foundational bar first, then layer on managed governance and full assurance. Tiers are cumulative.

Tier 1 — Foundational
8 controls · minimum production bar

The core safety and security controls that prevent the highest-severity, fastest-moving failures. The minimum bar for any production agent.

kill-switch · autonomy boundary · human oversight · content safety · agent identity · injection defence · observability · oversight
Tier 2 — Managed
18 cumulative · material systems

Adds governance, alignment and proactive security with defined evidence. Appropriate for material or customer-facing systems.

+ provenance · change · evaluation · threat modeling · security testing · labelling · monitoring · memory · multi-agent
Tier 3 — Assured
25 total · high-risk readiness

The full catalog with continuous conformance and third-party attestation. EU AI Act high-risk readiness.

+ interpretability · explainability · third-party · training · drift · lifecycle · carbon
